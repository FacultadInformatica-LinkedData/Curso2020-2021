Analyzing Dataset

Types of sample / magnitude that we can treat, that is, those that have in common.


Fields found at dataset 1 (2010 to February 2020) and dataset 2 (March to June 2020)
[The precise details of the measurements are located in this guide]

* Sulfur dioxide (SO2)
* Carbon monoxide (CO)
* Nitrogen dioxide (N02)
* Particulate matter 2.5 (PM2.5)
* Particulate matter 10 (PM10)
* Ozone (O3)

* Station / Sampling point: it is a numerical code that corresponds to a place: 28079001
* Date: DD / MM / YYYY the sample was taken.
   - In dataset 1 there is one row per month, with one column for each day.
   - In dataset 2 there is a row for each day.
* Validity (dataset 1)
   - Only the data with the V are valid, so we will have to filter them.

Defining Resource Naming Strategy

The following URI will be used to access all the data that we will use for our ontology:
- http://www.semanticweb.org/group16/ontologies/air-quality


I.e., the URI corresponding to the measurement class. We've chosen hash (#) because it is a periodic resource:
- http://www.semanticweb.org/group16/ontologies/air-quality#Measurement


In the following URI we would find the stations where the measurements are made, which will belong to a specific district, and which will be linked to real location places:
- http://www.semanticweb.org/group16/ontologies/air-quality#Station
